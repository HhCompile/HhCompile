# 脚手架工作流

> 模板项目快速搭建

## 🎯 适用场景

- 新项目启动
- 新功能模块（前端组件库 / 后端服务）
- 团队规范统一

## 📋 流程

```
1️⃣ 选型：搜索成熟骨架
2️⃣ 评估：多视角打分
3️⃣ 改造：克隆 + 删减 + 改名
4️⃣ 验证：能跑通最小流程
5️⃣ 沉淀：写入团队文档
```

## 🔧 详细步骤

### 1️⃣ 选型
- 搜索 GitHub / awesome 列表
- 优先选活跃维护的
- 看 demo / docs 是否齐全

### 2️⃣ 评估（多视角）
| 维度 | 评估点 |
|---|---|
| 安全性 | 依赖漏洞、权限模型 |
| 扩展性 | 是否支持插件化 |
| 相关性 | 跟项目匹配度 |
| 规划性 | 路线图清晰度 |

### 3️⃣ 改造
- 删除不需要的代码（不要留作"以备后用"）
- 重命名为项目规范
- 调整配置文件（lint / test / build）

### 4️⃣ 验证
- 跑通 README 中的 demo
- 跑通 CI
- 部署一次（验证完整流程）

### 5️⃣ 沉淀
- 记录"我们做了什么改造"
- 后续项目可复用

## 💬 提示词模板

```
我要做 [项目类型]，请推荐 3 个最匹配的脚手架：

要求：
- 活跃维护（最近 3 个月有 commit）
- 文档齐全
- 社区生态好
- 支持 TypeScript
```
