# 架构设计工作流

> 多视角分析决策

## 🎯 适用场景

- 大决策（选框架、拆服务、定协议）
- 长期影响的技术选型
- 跨团队规范

## 📋 流程

```
1️⃣ 收集：需求 + 约束 + 现状
2️⃣ 多视角：5 个角色同时审
3️⃣ 列方案：3+ 候选方案
4️⃣ 打分：多维度对比
5️⃣ 决策：选最优 + 记录 trade-off
```

## 🔧 5 个视角

| 角色 | 关注点 |
|---|---|
| **架构师** | 长期可维护性 / 一致性 |
| **资深工程师** | 实现难度 / 团队能力匹配 |
| **安全专家** | 攻击面 / 权限模型 / 数据合规 |
| **产品/业务** | 用户体验 / 业务灵活性 |
| **运维** | 部署复杂度 / 监控 / 成本 |

## 🔧 多维度打分

每个方案在以下维度打分（1-5）：

| 维度 | 权重 |
|---|---|
| 性能 | 25% |
| 可维护性 | 20% |
| 安全性 | 20% |
| 团队匹配度 | 15% |
| 成本 | 10% |
| 可扩展性 | 10% |

## 💬 提示词模板

```
我需要做 [决策]：

需求：
[详细]

约束：
[性能 / 成本 / 时间 / 团队]

请：
1. 列 3+ 候选方案
2. 用 5 个视角（架构师/工程师/安全/业务/运维）评估
3. 多维度打分
4. 推荐最优 + 解释 trade-off
```
