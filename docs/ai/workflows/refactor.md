# 重构工作流

> 大文件拆分 / 命名规范化 / 模式抽取

## 🎯 适用场景

- 接手"祖传"大文件（>800 行）
- 模块边界模糊的代码
- 重复出现的相似代码

## 📋 流程

```
1️⃣ 扫描：识别"坏味道"
2️⃣ 排序：按风险/收益排
3️⃣ 隔离：保证不破坏现有行为（测试先行）
4️⃣ 拆分：按功能/领域切分
5️⃣ 验证：所有测试通过
```

## 🔧 坏味道清单

| 坏味道 | 处理方式 |
|---|---|
| 巨型函数 (>50 行) | 拆分为多个小函数 |
| 巨型文件 (>800 行) | 按领域切分文件 |
| 重复代码 | 提取公共函数 |
| 长参数列表 (>4 个) | 用对象封装 |
| 命名随意 (a/b/aa) | 改语义化命名 |
| 散弹枪修改 (一处改 8 处) | 集中配置 / 循环 |

## 💬 提示词模板

```
请识别以下文件的重构机会：
- 超过 50 行的函数
- 重复模式
- 命名不清晰的变量
- 难以测试的代码段

[文件内容]
```

## 📌 注意事项

- **永远先有测试**：没有测试的重构是赌博
- **小步走**：一次只改一个点
- **Git 提交粒度**：每个重构点一个 commit
- **行为不变**：重构 ≠ 改功能
