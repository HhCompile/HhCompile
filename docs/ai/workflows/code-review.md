# 代码审查工作流

> 5 步走查流程，Claude 辅助

## 🎯 适用场景

- 日常 PR 审查
- 个人代码 review
- 上线前自查

## 📋 流程概览

```
1️⃣ 准备：拉取 diff + 上下文
2️⃣ 一遍过：宏观结构 + 命名
3️⃣ 二遍查：性能 + 安全 + 边界
4️⃣ 三遍深：可维护性 + 复用
5️⃣ 总结：分级反馈（CRITICAL / HIGH / MEDIUM / LOW）
```

## 🔧 详细步骤

### 1️⃣ 准备
- 拉取 PR diff
- 准备相关模块上下文
- 明确审查目标（性能 / 安全性 / 可读性 / 业务正确性）

### 2️⃣ 一遍过（结构 + 命名）
- 模块边界是否清晰
- 函数 / 变量命名是否自解释
- 是否有重复代码

### 3️⃣ 二遍查（性能 + 安全 + 边界）
- N+1 查询 / 大循环
- 输入校验、注入风险
- 边界值、空值、异常路径

### 4️⃣ 三遍深（可维护性 + 复用）
- 是否符合现有项目模式
- 是否需要提取工具函数
- 单元测试是否覆盖

### 5️⃣ 总结
- 用 4 级标注反馈：🔴 CRITICAL / 🟠 HIGH / 🟡 MEDIUM / 🟢 LOW
- 给具体修改建议（不是"这不好"，而是"改成 X 更好"）

## 💬 提示词模板

```
请审查以下 diff，重点关注：
- 性能问题
- 边界条件
- 命名一致性
- 测试覆盖

[diff 内容]
```
