# 最佳实践总结

> 跨工作流经验沉淀

## 🎯 通用原则

### 1. **明确意图，再动手**
- 写代码前先说清"为什么这样做"
- Claude 给的方案，先理解再采纳

### 2. **小步走，快反馈**
- 每次提交 < 200 行 diff
- 频繁运行测试
- 及时 commit

### 3. **测试先行**
- 没有测试的代码 = 借来的代码（迟早要还）
- 写测试就是写"使用说明"

### 4. **可读性 > 巧妙**
- 90% 的时间在读代码
- 一行 if 比三元表达式更易读

### 5. **留白比塞满好**
- 函数留 buffer
- 模块留边界
- PR 留 review 时间

## 🤖 Claude 协作心法

### ✅ 推荐用法
- **从零到一**：脚手架、模板、配置
- **从一到 N**：重复模式、批量操作
- **复杂查询**：跨文件搜索、模式识别
- **学习辅助**：解释陌生代码、对比方案

### ⚠️ 慎用
- **架构决策**：必须自己拿主意
- **核心算法**：自己懂才能调
- **安全相关**：必须人工 review
- **业务逻辑**：自己懂业务

### ❌ 不要用
- 一字不改的"AI 生成代码"
- 不理解就 commit 的方案
- 跳过测试的"省时间"

## 📊 量化收益

| 场景 | 不用 Claude | 用 Claude | 提升 |
|---|---|---|---|
| 写组件 | 2h | 30min | 4x |
| 调 bug | 1-2h | 20-40min | 3x |
| 写测试 | 1h | 20min | 3x |
| 文档 | 1h | 15min | 4x |
| 脚手架 | 1 周 | 半天 | 14x |

## 🛡️ 风险防控

- **代码所有权**：所有 AI 代码必须理解
- **安全审计**：AI 给的鉴权代码必须人工 review
- **依赖管理**：AI 推荐的库必须查活跃度
- **回滚准备**：重要改动要可回滚
